# AGENTS.md

## Global Rules

### Design First
- 对于中大型功能改动，先补充或更新设计文档，再进入具体实现。
- 设计文档确认前，不直接开始大范围编码、联调或重构。
- 如果改动会影响后端接口、前端交互、数据模型、权限规则、部署方式或测试策略，必须先写设计文档。
- 只有用户明确要求跳过设计文档，或者改动足够小且不影响设计边界时，才可以直接实现。

### Design Document Minimum Checklist
- 背景与现状
- 目标与非目标
- 影响范围
- 数据模型或权限模型
- 后端实现方案
- 前端影响
- 测试与回归方案
- 风险与分阶段落地建议

### Delivery Sequence
- 先写设计文档
- 再补或更新测试方案
- 最后进行实现、联调和发布
